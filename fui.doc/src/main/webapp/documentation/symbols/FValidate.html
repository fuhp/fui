Validate

FValidate在jquery.validate的基础上进行了扩展，主要扩展的功能：① 含有隐藏域的表单校验；② 支持类似于ext的qtip提示方式。

依赖组件

  • 无

适用范围

适用于 需要对表单校验，并且form中存在 FUI含有隐藏域的表单组件，比如NumberField、combo、calendar的情况下。

约束

  • 由于集成了jquery.validate，FUI只能保证经过验证过的功能是可以使用并建议使用的，其他的由于jquery.validate的缺陷造成问题建议采用规避的方式解决；

注意事项

  • 无

属性

名称 类型 描述 默认值
errorClass String

指定错误提示标签的class名称，此class也将添加在校验的元素上面。
说明：错误提示的样式包括输入框的样式以及提示信息的样式。该属性一般情况下不需要设置，使用默认值即可。除非用户自定义错误提示的样式。该样式具体制定详见demo。


示例： 
$(".selector").FValidate({
     errorClass: "invalid"
  })
 'error'
errorContainer Selector

包含错误提示信息的容器，根据校验结果隐藏或者显示错误容器。其中该错误提示信息是由用户在页面设计的时候填写的内容。
与 errorLabelContainer 属性的区别是这个属性提示的错误信息是静态的，即为用户在页面设计时填写的内容。


示例： 
$("#myform").FValidate({
     errorContainer: "#messageBox1, #messageBox2"
     //可以配置多个容器，这里的messageBox2元素没有被包装处理，只是错误发生的时候显示和隐藏此元素。
  })
 $( [] )
errorElement String

指定错误信息的html标签名称
说明：该属性只有当用户设置errorPlacement属性时才起作用。


示例： 
$(".selector").FValidate({
     errorElement: "em"
  })
 'div'
errorLabelContainer Selector

设置统一存放错误信息的容器，根据校验结果隐藏或者显示错误容器。校验框架会将所有的错误提示信息存放到该容器下。


示例： 
$("#myform").FValidate({
     errorLabelContainer: "#messageBox1 ul"
     //messageBox为容器的id
  })
 $( [] )
errorModel String

错误信息提示的模式。默认模式为“under”。有两种模式，“under”模式为：错误信息显示在输入框的下方。“qtip”模式为：错误信息悬浮显示。


示例： 
$(".selector").FValidate({
     errorModel: "qtip"
  })
 'under'
errorPlacement Function

定制错误信息显示的回调方法，用于自定义错误提示格式。该方法有两个参数，第一个参数是错误信息的元素，第二个是触发校验错误的源元素。FCombo组件和FNumberField组件第二个参数为隐藏域元素。


示例： 
$("#myform").FValidate({
    errorPlacement: function(error, element) {
       // 用户自定义操作
       error.appendTo( element.parent("td").next("td") );
     }
   })
 无
focusCleanup Boolean

获得焦点的时候是否清除错误提示，这种清除是针对所有元素的，
如果设置为true，则必须将focusInvalid设置为false，否则将没有校验效果。


示例： 
$(".selector").FValidate({
     focusInvalid: false, //必须设置
     focusCleanup: true
  })
 false
focusInvalid Boolean

校验错误的时候是否将聚焦元素。该属性如果为true，校验出错时会将光标聚焦到错误处。
说明:使用该属性时，提交按钮类型必须是"submit"类型。


示例： 
$(".selector").FValidate({
     focusInvalid: false
  })
 true
groups JSON

错误消息分组,如果没有设置errorPlacement，则分组内的元素出现错误时只在第一个元素后面显示错误消息， 如果设置了errorPlacement，则可以在errorPlacement回调中定义显示位置


示例： 
$("#myform").FValidate({
    groups: {
      username: "fname lname"
    },
    errorPlacement: function(error, element) {
       if (element.attr("name") == "fname"
                   || element.attr("name") == "lname" ){
         error.insertAfter("#lastname");
       }else{
         error.insertAfter(element);
       }
     },
   }) //将fname和lname的错误信息统一显示在lastname元素后面
ignore String

校验时忽略指定的元素，可以配置需要校验的元素id和样式名称等jquery识别的选择器。
说明：该属性值不能为 “：hidden”，否则会导致FCombo、FNumberField组件的校验无效。建议为复杂的选中器，用过不匹配个别标签。


示例： 
$("#myform").FValidate({
     ignore: ".ignore"
     //此处还可以配置input[type='password']、#id等jquery的选择器
  })
 null
messages JSON

键值对的校验错误信息.键是元素的name属性，值是错误信息的组合对象。


示例： 
$("#formId").FValidate({
 rules: {
   name: {
     required: true,
     minlength: 2
   }
 },
 messages: {
   name: {
     required: "We need your email address to contact you",
     minlength: jQuery.format("At least {0} characters required!")
     //这里的{0}就是minlength定义的2
   }
 }
})
onfocusout Boolean

在blur事件发生时是否进行校验，如果没有输入任何值，则将忽略校验。


示例： 
$(".selector").FValidate({
     onfocusout: false
  })
 true
onkeyup Boolean

在keyup事件发生时是否进行校验。


示例： 
$(".selector").FValidate({
     onkeyup: false
  })
 true
onsubmit Boolean

是否在提交时校验表单，如果设置为false，则提交的时候不校验表单，
但是其它keyup、onblur等事件校验不受影响.


示例： 
$(".selector").FValidate({
     onsubmit: false
  })
 true
submitHandler Function

定制校验通过后表单提交前的回调方法，用来替换默认提交，一般是Ajax提交方式需要使用到。


示例： 
$(".selector").FValidate({
     submitHandler: function(form) {
      $(form).ajaxSubmit(); //校验通过之后调用ajaxSubmit提交表单
     }
  })
 undefined
success String

指定校验成功没有任何错误后加到提示元素上面的样式名称
和validClass的区别是它只加在提示元素上面，而不对校验的对象做任何变动。


示例： 
$(".selector").FValidate({
     success: "valid"
  })
 无
validClass String

指定校验成功没有任何错误后加到元素的class名称


示例： 
$(".selector").FValidate({
     validClass: "success"
  })
 'valid'

方法

名称 参数 描述
element (element)

校验选中的element

参数:
  • :element
    • 示例： 
     $("#myform").FValidate().element( "#myselect" );
    form ()

    触发表单校验

    示例： 
     $("#myform").FValidate().form()
    FValidate ()

    创建表单的校验对象

    示例： 
      $("#myform").FValidate({
     //options
  });
    numberOfInvalids ()

    统计没有通过校验的元素个数

    示例： 
    var validator = $("#myform").FValidate();
return validator.numberOfInvalids();
    resetForm ()

    重置表单，调用此方法将去掉所有提示信息

    示例： 
     var validator = $("#myform").FValidate();
 validator.resetForm();
    rules ()

    针对选中的元素，动态添加删除校验规则的方法，有rules( "add", rules ) 和rules( "remove", [rules] )两种

    示例： 
     $('#username').rules('add',{
      minlength:5,
      messages: {
          minlength: jQuery.format("Please, at least {0} characters are necessary")
      }
  });

$("#myinput").rules("remove", "min max"); //remove可以配置多个rule，空格隔开
    showErrors (Object)

    添加并显示提示信息,其中示例中的firstname和firstname1为form表单下面的一个表单组件的name属性值。

    参数:
  • :Object
    • 示例： 
    var validator = $("#myform").FValidate();
validator.showErrors({"firstname": "请重新选择！",'firstname1':'该字段输入信息有错误！'});
    valid ()

    检查表单是否通过校验

    示例： 
      $("#myform").FValidate();
  $("a.check").click(function() {
    alert("Valid: " + $("#myform").valid());
    return false;
  });
